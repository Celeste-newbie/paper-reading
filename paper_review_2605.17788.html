Uncertainty-Aware Adaptive Recommendation
across User Lifecycle

用户全生命周期的不确定性感知自适应推荐
作者：Bob Junyi Zou (Stanford) · 字节跳动团队
平台：抖音直播 (大规模线上验证)
发表：KDD '26, Jeju Island
arXiv：2605.17788v2
📝
摘要

推荐系统的核心挑战在于如何为低活跃用户（LAU）提供可靠推荐的同时， 为高活跃用户（HAU）提供多样化内容。 本文的核心钥匙是模型不确定性量化——它既能预警低质量推荐、阻止低活用户流失， 又能识别内容探索机会、为高活用户拓宽兴趣边界。 框架在字节跳动直播平台大规模部署，低活用户留存（HLT7）和观看质量（VWR）显著提升， 高活用户内容多样性和品类覆盖率大幅增加。

💡
核心洞察（最大创新点）
🔀
同一个"不确定性信号"，截然相反的两种用法
传统方法把不确定性一律视为"坏事"需要消除。本文的关键洞察是： 对 LAU，高不确定性 = 风险（应该压制）；对 HAU，高不确定性 = 探索机会（应该利用）。 这一对偶视角让同一套不确定性量化框架驱动出截然相反的策略，是整篇论文最核心的思想贡献。
⚙️
洞察得以落地的技术保障：统一不确定性量化框架
大多数工业级推荐模型只输出一个标量预测值 fΘ(x)，对自身预测的可靠性毫无感知。本文为两类主流模型分别设计了不确定性量化方法，统一成一套低延迟、可落地的框架：
• 方法 A（EPE Critic）：针对确定性点估计模型，输出 Upoint(x)
• 方法 B（Empirical Bayes）：针对概率性模型，输出 Uprob(x)
两者共同驱动贡献二中的 LAU/HAU 策略——是对偶洞察从思想走向工程的关键一步。
LOW-ACTIVE USERS · LAU

🛡️ 风险规避：Deboosting

交互历史稀疏，模型对这类用户预测最不可靠。一旦推错内容，用户极易永久流失（churn）。

高不确定性内容 → 降分（deboost） → 过滤掉高风险推荐

两大技术贡献详解
①
统一不确定性量化框架
Production-Ready Unified Uncertainty Quantification Framework
EPE Critic Empirical Bayes 低推理延迟

如何让工业级推荐系统"知道自己有多不确定"？本框架的答案是：分情况讨论。针对确定性模型（只输出标量）用 EPE Critic 间接度量；针对概率性模型（输出分布参数）用 Empirical Bayes 直接提取——两条路都在单次推理内完成，无额外延迟。

▍不确定性的形式定义

对每个 (user, context) 输入 x ∈ 𝒳，令 fΘ(x) 为推荐模型预测值，Θ 为每日训练后的参数随机变量。不确定性来源于两类：

认知不确定性
Epistemic
→
模型容量不足 / 训练数据稀疏 / 在线 serving 特征漂移 → 可随数据增加而减少
随机不确定性
Aleatoric
→
用户行为本身的内在随机性 𝕍(Y | X=x) → 不可减少，即使模型完美

本文聚焦于认知不确定性——直接反映模型对当前 (user, item) 交互模式的掌握程度。下方方法 A / 方法 B 分别量化这一信号，最终输出 Upoint(x) 和 Uprob(x) 供贡献二调用。

▍方法 A vs 方法 B：适用场景与原理对比
①-A EPE Critic 网络
适用模型确定性点估计模型（CTR、时长回归）
不确定性来源历史泛化误差（跨 checkpoint 波动）
训练方式辅助 Critic 网络，MSE 回归，冻结主模型
推理代价单次前向，无额外延迟
优势场景即时风险规避（VWR 提升更显著）
输出信号Upoint(x) = gφ(zΘ(x))
VS
①-B Empirical Bayes 先验头
适用模型概率性输出模型（带分布参数输出）
不确定性来源参数先验分布的方差（Type-II MLE）
训练方式最大化边缘似然，学习超参 γ，θ 保留分布
推理代价单次前向，Beta 方差解析计算
优势场景兴趣探索稳定性（Show Tag/U 更优）
输出信号Uprob(x) = αβ / [(α+β)²(α+β+1)]

两者正交互补：EPE 捕获 serving 系统偏差，Bayes 捕获参数分布方差。联合使用在 HLT7（+0.0577%）和 Show Tag/U（+2.1%）上均超过任一单独使用。

①-A
方法 A：EPE Critic 网络
Expected Prediction Error Critic — for Deterministic Point-Estimate Models
CTR 预测器 Decoupled Training Monte Carlo 近似

对于只输出点估计 fΘ(x) 的模型（如 CTR 预测器、观看时长回归器），模型本身没有任何不确定性输出。本方法训练一个辅助 Critic 网络，让它学会"预测主模型会犯多大的错"。

▍EPE 定义与偏差-方差-噪声分解

定义输入特定的期望预测误差：

EPE(x) = 𝔼Θ 𝔼Y|X=x [ ℓ(Y, fΘ(x)) ]

取平方损失，EPE 可做经典偏差-方差-噪声分解：

fΘ(x)
某次具体训练（参数 Θ）的预测值；Θ 随训练集采样/初始化等随机因素变化
f̄(x) = 𝔼Θ[fΘ(x)]
所有训练随机性的期望预测均值（理论量）；fΘ 围绕 f̄ 的波动幅度 = 方差 = 认知不确定性
μ(x) = 𝔼[Y|X=x]
真实条件均值（未知）；f̄ 与 μ 的系统性偏移 = 偏差²
▶ 展开完整推导

一次性加减 f̄(x) 和 μ(x)，把 (Y − fΘ(x)) 拆成三段，展开后三个交叉项均为 0：

EPE(x)= 𝔼Θ 𝔼Y|x [ (Y − fΘ(x))² ]
= 𝔼Θ 𝔼Y|x [ (Y−μ(x)) + (μ(x)−f̄(x)) + (f̄(x)−fΘ(x)) ]²
三个交叉项消去：① 𝔼Y|x[Y−μ(x)]=0（零均值噪声）  ② 𝔼Θ[f̄(x)−fΘ(x)]=0（f̄ 是 fΘ 的期望）  ③ 两常数项与随机项协方差=0
= 𝕍(Y|X=x)  +  (μ(x)−f̄(x))²  +  𝔼Θ[(fΘ(x)−f̄(x))²]
数据噪声 ε²
𝕍(Y|X=x) · 不可消除 + 偏差² Bias²
(f̄−μ)² · 换更强模型可减 + 方差 Variance
𝔼Θ[(fΘ−f̄)²] · 认知不确定性

为什么 EPE 可代理认知不确定性：论文只在同一用户群（LAU）内部比较 EPE，此时 ε² 近似相同，EPE 差异主要反映 Variance 差异，即认知不确定性高低。

EPE(x) = (f̄(x)−μ(x))² + 𝔼Θ[(fΘ(x)−f̄(x))²] + 𝕍(Y|X=x)
模型偏差² 模型方差（认知不确定性） 数据噪声（随机不确定性）
▍Critic 网络的训练

直接计算 EPE 需对 Θ 采样，代价极高。转而训练辅助 Critic 网络 gφ 直接预测 EPE(x)：

Step 1
构建监督信号：每日 checkpoint Θ(1),...,Θ(K)。用前一天参数评估当天样本，计算实际泛化误差：
ej(k) = ℓ(yj(k), fΘ(k−1)(xj(k)))
用 k−1 的参数评估第 k 天样本，避免信息泄漏，构成 EPE 的 Monte Carlo 近似。
Step 2
多视角输入表示：拼接原始特征 + 主模型中间 embedding + 主模型输出：
zΘ(x) = [ x ‖ hΘ(x) ‖ fΘ(x) ]
Step 3
解耦训练：训练 Critic 时冻结主模型，防止反馈回路。目标：MSE 回归
minφ 𝔼[ (gφ(zΘ(x)) − e)² ]　→　Upoint(x) ≡ gφ(zΘ(x))
①-B
方法 B：Empirical Bayes 先验头
Type-II Maximum Likelihood — for Probabilistic Output Models
Beta 先验 全变差分解 单次推理

对于输出分布参数的概率性模型，本方法通过 Type-II 最大似然（Empirical Bayes） 训练，使模型天然保留对参数 θ 的不确定性。

▍神经网络输出输入自适应先验

令神经网络（参数 Θ）为每个输入 x 输出先验超参 γΘ(x)，先验为：

θ | x ∼ p(θ | γΘ(x))

边缘预测分布通过积分消去 θ：

p(y | x) = ∫ p(y | θ) · p(θ | γΘ(x)) dθ

训练目标（Type-II MLE）：

Θ* = argmaxΘ   Σi log ∫ p(yi | θ) · p(θ | γΘ(xi)) dθ

注：γ 不是手动设定的固定先验，而是由神经网络从数据中"学"出来（Empirical = 经验性）。每个 (user, item) 对有自己独特的先验，故称"输入自适应先验"。

▶ Type-I MLE vs Type-II MLE：为什么 Type-II 能保留不确定性？

两种方法对潜在参数 θ 的处理方式截然不同，这决定了能否保留不确定性：

Type-I MLE（普通最大似然）
优化目标：直接最大化观测数据的似然
θ* = argmaxθ   p(y | θ)
直接学习 θ 的点估计 θ*，训练完成后 θ 被固定为一个确定值。
不确定性完全丢失——无法知道模型对这个预测"有多没把握"。
❌ 输出：单个点 θ*，不确定性归零
→
Type-II MLE（Empirical Bayes）
优化目标：最大化边缘似然（积分掉 θ）
γ* = argmaxγ   ∫ p(y | θ) · p(θ | γ) dθ
不学习 θ 的点估计，而是学习先验超参 γ——让先验分布去"解释数据"。θ 始终是随机变量，其分布 p(θ|γ) 的方差就是认知不确定性。
✅ 输出：先验分布 p(θ|γ)，不确定性被保留
完全贝叶斯对 γ 也设先验，三层推断 → 理论最完整，计算不可行
Empirical Bayes（本文）用数据估计 γ，两层结构 → 保留不确定性，可微可训练
普通 MLE直接学 θ 点估计，一层 → 最简单，不确定性全失
▍全变差公式：提取认知不确定性

由于 θ 保有分布（而非点估计），预测方差可通过全变差公式（Law of Total Variance）在单次推理中解析分解：

Var(Y|X=x) = 𝔼θ|x[Var(Y|θ)] + Varθ|x(𝔼[Y|θ])
随机不确定性 Aleatoric（用户行为内在随机性，不可消除） 认知不确定性 Epistemic ← 本文取此项

只取认知项作为不确定性信号：Uprob(x) = Varθ|x(𝔼[Y|θ])

▍具体实现：Beta-Bernoulli 共轭，解析解

以 CTR 建模为例（Bernoulli 似然 + Beta 先验共轭对），模型输出两个 logit，经 softplus 参数化：

α(x) = 1 + softplus(u(x)),   β(x) = 1 + softplus(v(x))
θ | x  ∼  Beta(α(x), β(x))

Beta 先验方差（即认知不确定性）有解析解，单次推理直接计算：

Uprob(x)  =  Varθ|x(𝔼[Y|θ])  =  α(x) · β(x) (α(x)+β(x))² · (α(x)+β(x)+1)
②
LAU / HAU 不确定性对偶用法框架
Dual Usage: Risk-Averse Deboosting (LAU) & UCB Exploration (HAU)
LAU → 风险过滤 HAU → 探索驱动 同一信号·相反策略

贡献一产生的不确定性信号 Upoint / Uprob 对不同用户群体有截然相反的含义：对低活跃用户（LAU）是风险信号（→ 降分过滤），对高活跃用户（HAU）是探索机会（→ UCB 加分）。这种对偶视角是整篇论文最核心的思想贡献。

②-A
低活跃用户（LAU）：风险规避 Deboosting
Risk-Averse Deboosting for Low-Active Users
HLT7 +0.057% VWR +0.512% 不确定性 = 风险

低活跃用户（LAU）定义为月活跃观看天数 < 7 天。其交互历史极度稀疏，模型预测可靠性最低——一旦推荐失当极易永久流失（churn）。本策略将贡献一的不确定性信号用作风险过滤器，主动压制高不确定性推荐。

▍Step 1：经验阈值校准（无参数假设）

在独立 hold-out 校准集 𝒟cal 上，对每个样本计算两类不确定性分值，取第 q 分位数（默认 q = 0.95）作为置信阈值：

τpoint = Quantileq({Upoint(xj) : (xj, yj) ∈ 𝒟cal})
τprob = Quantileq({Uprob(xj) : (xj, yj) ∈ 𝒟cal})

关键优势：无需对不确定性分布做任何参数假设，直接从真实数据分布中提取阈值，对 distribution shift 天然鲁棒。

▍Step 2：风险判定（保守的 OR 逻辑）

对 LAU 的每个候选 item，同时评估两类不确定性，只要任意一类超阈值即认定为"高风险"推荐：

𝕀risky(u, i) = 𝕀[ Upoint(xu,i) > τpoint OR Uprob(xu,i) > τprob ]

选用 OR 而非 AND 的原因：两种不确定性来源不同（EPE 捕获历史泛化误差，Bayes 捕获参数分布方差），互补而非冗余，保守地取并集可最大化召回高风险样本。

▍Step 3：降分与排序

对高风险 item 施加固定幅度的降分，并对最终分值重新排序：

rfinalu,i = ru,i − D · 𝕀risky(u, i)

Rankfinal(u) = argsort({ rfinalu,i : i ∈ ℐ(u) })

超参数 D（降分幅度）和 (τpoint, τprob) 通过线上 A/B 实验联合调优。

▍直觉解释：为什么 Deboosting 能提升留存？
📉
传统系统：高置信度预测 ≠ 高质量推荐（模型校准差）→ 对 LAU 推高风险内容 → 用户不满 → 流失
✅
本文策略：在高不确定性样本上主动"退让"，宁可少推，也要推可靠的 → 降低 LAU 第一批次体验中的负样本率 → 用户留存
🔬
验证：Live Watch Time 轻微下降（−0.16%）但 HLT7 显著提升，HLT-efficiency = 0.0549，说明用"少量观看时间"换取了"高质量留存"，是系统性优化而非简单截断。
②-B
高活跃用户（HAU）：不确定性驱动 UCB 探索
Uncertainty-Driven UCB Exploration for High-Active Users
Show Tag/U +2.1% UCB 原则 不确定性 = 机会

高活跃用户（HAU，月活 ≥ 7 天）拥有丰富交互历史，模型对其预测通常较可靠，但系统存在"越推越窄"的马太效应。本策略将贡献一的不确定性信号反转为探索驱动力——越不确定，越值得探索。

▍理论基础：UCB 原则与探索-利用权衡

UCB（Upper Confidence Bound）来自多臂老虎机（Multi-armed Bandit）理论。经典 UCB1 算法对每个臂的打分为：

UCBt(a) = X̄a + √(2 ln t / na)

其中 X̄a 为当前估计均值（利用），√(2 ln t / na) 为置信区间半宽（探索奖励）。对观察次数少（na 小）的臂，探索奖励大，鼓励优先尝试。

本文将此原则迁移至推荐排序：认知不确定性高的 item 对应"模型尚未充分理解用户对此内容的偏好"，类比 na 小的臂，应给予探索加分。

▍HAU 不确定性聚合与排序加分

对 HAU，将两类不确定性取最大值作为探索信号（保守取上界，确保任一维度的探索机会均被捕获）：

UHAU(xu,i) = max(Upoint(xu,i), Uprob(xu,i))

最终排序分加入探索奖励：

rfinalu,i = ru,i + ω · UHAU(xu,i)

其中 ω > 0 为探索权重，通过 A/B 实验调节，平衡短期满意度与长期多样性。

▍认知不确定性 vs 随机不确定性：为什么只用认知部分？
🧠
认知不确定性高：模型对该用户-内容对学习不充分 → 探索价值高，推荐后可以收集反馈、改善模型 → 应该加分
🎲
随机不确定性高：用户行为本身随机波动大 → 即使推荐了，收集到的反馈信噪比低，对模型提升有限 → 不应作为探索依据

这也是为什么对概率性模型，本文专门通过全变差分解剥离出认知部分 Uprob，而非直接使用总预测方差。

大规模工业验证与消融分析

论文在字节跳动直播平台的真实生产流量上进行了为期 14 天的 A/B 实验，并设计了 5 组消融变体，严格区分"不确定性量化质量"与"排序策略本身"对结果的贡献。

▍离线验证：不确定性估计质量评估

由于认知不确定性没有 ground truth，论文采用三类间接评估指标，从不同角度验证"不确定性分值是否真的能区分高风险样本"：

AURC
Area Under Risk-Coverage Curve（风险-覆盖曲线下面积）
直觉
对测试集按不确定性从高到低排序，逐步"拒绝"高不确定性样本，观察剩余样本的平均预测误差如何变化。如果不确定性分值有效，拒绝高分样本后误差应快速下降。
含义
AURC = 拒绝曲线下面积。越低越好——说明模型能精准识别高风险样本，少量过滤即可大幅降低误差。
本文结果
EPE Critic 0.0157 < MC Dropout 0.0219 < Multi-head 0.0241（越低越好）
类比：AURC 之于不确定性，相当于 AUC 之于分类——衡量"排序质量"而非绝对值。
Spearman
Spearman 秩相关系数（ρ）
直觉
理想的不确定性估计应与实际预测误差"单调一致"——不确定性高的样本，真实误差也应更大。Spearman ρ 衡量这种排序一致性，不要求线性关系。
含义
ρ ∈ [-1, 1]，越接近 1 说明不确定性分值与误差的排序越吻合。ρ ≈ 0 意味着随机，ρ < 0 反向。
本文结果
本文方法 > 0.65；Multi-head Ensemble 0.276；MC Dropout 0.189（越高越好）
0.65 vs 0.189 的差距说明本文不确定性分值与真实误差的排序一致性远超 baseline，是 Deboosting/UCB 策略能奏效的直接依据。
趋势对齐
直播流年龄验证（Sanity Check by Stream Age）
直觉
刚开播的直播（历史交互 = 0）模型几乎没有任何信息，不确定性应最高；随着直播持续、积累互动，模型逐渐"了解"该直播，不确定性应单调下降。这是先验可验证的物理约束。
验证方式
按直播时长（stream age）分桶，对每桶计算平均 Upoint / Uprob，检验是否与直播年龄呈负相关，且走势与在线 RMSE 曲线高度一致。
本文结果
两类不确定性均通过验证：stream age 越大，Upoint / Uprob 越低，趋势与在线 RMSE 高度吻合。
这一指标是"领域知识 + 数据"的交叉验证，能排除"不确定性分值偶然高但无实际意义"的情况，增强工程可信度。

▍关键推论：Random Baseline 是"验伪"实验

论文专门设计了一组 Random Baseline——用均匀随机分值替代学习到的不确定性，执行同样的 deboosting/boost 操作。结果：

▍EPE 与 Bayes 互补性分析
EPE Critic 的优势
直接拟合历史泛化误差，对即时风险规避（VWR）更有效。能捕获在线 serving 的系统性偏差（特征 staleness、延迟等）。
Bayes 先验方差的优势
来自参数先验分布，对兴趣探索（Show Tag/U）更稳定。仅靠 Bayes，HAU Show Tag/U 已达 +0.9%，比 EPE 单独的 +1.0% 接近。

两者联合使用，在 HLT7（+0.0577% vs +0.0434%/+0.0468%）和 Show Tag/U（+2.1% vs +1.0%/+0.9%）上均超过任一单独使用，验证了两种信号的正交互补性。

⚠️
局限性与不足（论文自陈）
🔄

依赖稳定的日训练管线

EPE 的监督信号需要每日 checkpoint 对比，若训练管线不稳定或节奏不规律，EPE 样本质量会下降。

📐

Beta 先验可能过于简化

对于复杂多模态行为（如同时看视频、直播、购物的用户），Beta 分布先验可能不足以刻画真实的行为分布。

🎯

认知不确定性无 Ground Truth

评估只能间接进行（通过观察在线 MSE 的相关性），无法直接度量"真实的认知不确定性"。

⚖️

Deboosting 的长期权衡

压制高不确定性内容保护了短期体验，但同时减少了系统学习这些内容的机会，可能影响长期的模型迭代速度。

👥

LAU 内部异质性未细分

LAU 群体内部差异巨大（极端冷启动 vs. 轻度流失用户），当前框架未进行更细粒度的差异化处理。
